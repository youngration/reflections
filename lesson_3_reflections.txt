-何时你会想使用远程版本库，而不是将所有工作保存在本地？
一些描述性的东西、一个灵感/想法的记录等等。

-为何你希望始终手动地拉取更改，而不是让 Git 自动与你的远程版本库保持同步？
这样可以在我确认需要同步的时候再进行同步，否则可能引起不必要的错误。

-说明Fork、克隆和分支之间的不同。何时使用某种功能而不是另一种？
fork是github上的操作，将别人的代码库fork到自己的账号下。clone需要有本地计算机的参与，或者从远程clone代码库到本地，或者从本地clone代码库到本地另一处。branch则是版本库对提交的管理。

-拥有远程分支的上次已知状态的副本（存储在本地）有何好处？
标识出本地版本库在哪个提交后和远程不同步，这样可以在本地分支继续提交，合适的时候再fetch远程分支，然后和本地分支合并。

-如何在不使用 Git 或 GitHub 的情况下开展协同？哪种做法更容易？哪种做法更困难？
用svn，或者每天下班前大家把工作都拷贝到主管电脑上并由他管理。前者方便。后者困难。

-何时你会想在独立的分支中而不是直接在 master 中进行更改？两种方法各有何好处？
使用别人的代码时，或者多人合作完成工作时。独立分支一方面保证主分支的安全，另一方面允许多人协作，而直接在master进行更改会比较简单，不需要处理冲突。
